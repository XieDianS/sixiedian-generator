# 代码生成器平台设计说明

## 一、项目背景

在实际开发中，项目初始化、依赖集成、插件配置等工作，往往依赖**复制粘贴 + 手动修改**，效率低且容易出错。

本项目旨在开发一个**通用代码生成器平台**，支持将**开源项目、依赖或插件的使用方式**抽象为**可配置的代码生成器**，并通过 CLI 与平台进行分发和复用。

------

## 二、核心目标

- 将“人工修改代码”的过程**参数化、自动化**
- 支持 **完整项目初始化** 与 **功能/依赖级代码注入**
- 为开源作者提供 **生成器制作与分享平台**
- 为使用者提供 **一键生成 / 一键集成** 的开发体验

------

## 三、需求分析

### 1. 代码生成能力

1. 支持根据用户配置的参数，**动态生成代码文件**
2. 对于无需修改的文件，支持**原样复制**
3. 生成结果保持与原项目**一致的目录结构**
4. 支持文件内容、文件名、项目名的参数替换

------

### 2. 生成器制作能力（平台侧）

1. 用户可上传：
   - 完整项目源码（项目级生成器）
   - 单个依赖 / 插件 / 功能模块代码（功能级生成器）
2. 通过图形化方式：
   - 标记可配置参数
   - 指定模板文件与原样复制文件
   - 生成标准化的生成器描述文件

------

### 3. 生成器管理与分发

1. 用户可对自己的生成器进行：
   - 公开 / 私有 / 删除
2. 支持：
   - 收藏
   - 下载
   - 本地使用（CLI）
3. 支持生成器版本管理（可选）

------

## 四、整体开发流程

### 阶段一：生成器定义与 CLI 引擎

1. 选取一个开源项目或简单示例项目
2. 将其改造为**参数化代码生成器**
3. 实现 CLI 执行引擎：
   - 读取生成器定义
   - 收集参数
   - 生成目标代码

------

### 阶段二：生成器制作工具（Demo）

1. 提供 Web 图形界面
2. 支持上传源码并可视化配置参数
3. 自动生成生成器配置文件
4. 支持在线测试生成效果

------

### 阶段三：生成器平台

1. 提供生成器存储与分发服务
2. 支持生成器的权限管理（公开 / 私有）
3. 提供下载与 CLI 集成入口
4. 构建生成器生态

## 五、生成器的两种类型（关键设计）

### 1. 项目级生成器（Project Generator）

- 用于生成完整项目骨架
- 仅在项目初始化阶段使用
- 示例：
  - Vue 项目模板
  - Spring Boot 项目模板

------

### 2. 功能级生成器（Feature Generator）

- 用于为现有项目添加功能或依赖
- 可重复、可叠加使用
- 示例：
  - MyBatis 集成
  - JPA 集成
  - Redis / Swagger / 登录模块
